Module 1: Web Programming Basics

Understanding the Fundamentals of Web Architecture

Introduction

Web Programming Basics is the foundation of all web development. This module introduces you to the essential concepts that power the internet and web applications. You'll learn how browsers, servers, and protocols work together to deliver content across the globe.

Learning Outcomes

Upon completion of this module, you will be able to:

  • Explain the client-server architecture
  • Understand HTTP and HTTPS protocols
  • Describe how web browsers and servers communicate
  • Identify the key components of web standards
  • Explain the role of DNS and domain names
  • Understand web security basics

Client-Server Architecture

What is Client-Server Architecture?

Client-Server architecture is a network application architecture that separates tasks between clients (consumers of resources) and servers (providers of resources).

The Client:

  • Web browser (Chrome, Firefox, Safari, Edge)
  • Sends HTTP requests to the server
  • Displays the response to the user
  • Processes user interactions

The Server:

  • Listens for incoming requests
  • Processes the request
  • Sends back an HTTP response
  • Stores and manages data

HTTP Protocol

HyperText Transfer Protocol (HTTP)

HTTP is the foundation of data communication for the World Wide Web. It is a stateless, application-layer protocol.

HTTP Methods (Verbs):

  • GET: Request data from a server
  • POST: Submit data to a server
  • PUT: Update a resource on the server
  • DELETE: Remove a resource from the server
  • HEAD: Like GET but without the message body

HTTP Status Codes:

  • 1xx: Informational
  • 2xx: Success (e.g., 200 OK)
  • 3xx: Redirection (e.g., 301 Moved)
  • 4xx: Client Error (e.g., 404 Not Found)
  • 5xx: Server Error (e.g., 500 Internal Server Error)

Key Concepts

DNS

Domain Name System - Translates domain names to IP addresses so browsers can load internet resources.

IP Address

A unique identifier assigned to devices on the internet. Format: 192.168.1.1

Port Numbers

Communication endpoints. HTTP uses port 80, HTTPS uses port 443.

HTTPS

Secure version of HTTP with SSL/TLS encryption for safe data transmission.
